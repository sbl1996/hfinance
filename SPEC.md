# 个人资产管理记账系统

## 1. 项目概述
*   **项目定位**：供个人使用的聚合记账与投资管理 Web 应用（Mobile First）。
*   **核心痛点**：解决跨平台资产（存款/A股/港股/场外基金）统揽困难，免去手工计算市值与总资产的繁琐。
*   **核心机制**：**本金流转手动记账 + 投资品价格低频自动抓取 = 实时呈现净资产与盈亏明细**。

---

## 2. 产品功能需求

### 2.1 资产账户管理（手动维护）
*   **现金与理财**：支持多笔现金资产的增、删、改（如：招行卡余额、微信零钱、公积金）。
*   **负债录入**：支持信用卡、房贷等负债的增、删、改。
*   **投资持仓录入**：支持添加标的持仓。**重要规则**：无论A股港股，**持仓成本均以人民币（CNY）总额录入**（适配港股通直接扣除人民币的特性）。

### 2.2 数据可视化大盘 (Dashboard)
*   **资产总览卡片**：展示净资产、总资产、总负债、今日总盈亏、累计总盈亏。
*   **资产分布统计**：现金、投资、负债的占比饼图。
*   **持仓明细列表**：展示各个标的的名称、代码、持有数量、最新价、当前市值(CNY)、持仓收益率。
*   **盈亏日历**：按自然月展示日历网格，每日格子里展示当日的投资真实盈亏金额。
    1.  **UI规范**：正收益使用红色字体（如 +1,200），负收益使用绿色字体（如 -500），无收益/周末空白。点击具体日期可弹出当日盈亏明细（各标的分别赚/亏多少）。

### 2.3 行情与汇率抓取引擎 (Auto-fetch)
*   **抓取对象**：A股ETF每日收盘价、港股收盘价(HKD)、场外基金单位净值、**港元/人民币(HKD/CNY)实时汇率**。
*   **触发机制**：
    1.  **定时自动**：交易日 16:30 后，后端自动执行一次全量更新。
    2.  **手动触发**：前端提供「强制刷新」按钮，调用接口立即抓取最新价格。

---

## 3. 业务规则与计算逻辑

### 3.1 资产计算公式
*   **港股市值 (CNY)** = `港股最新价(HKD) × 持仓数量 × HKD/CNY最新汇率`
*   **A股/场外基金市值 (CNY)** = `最新价/净值 × 持仓数量`
*   **单标的盈亏金额** = `当前市值(CNY) - 录入的持仓总成本(CNY)`
*   **个人总资产** = `∑所有现金 + ∑所有持仓市值(CNY)`
*   **个人净资产** = `个人总资产 - ∑所有负债`
*   **单日盈亏计算公式（剥离现金流干扰）**：
    *   为了避免用户存取现金、买卖调仓导致总资产变动被误认为盈亏，单日盈亏仅计算**持仓市值的自然波动**。
    *   `某标的今日盈亏(CNY)` = `(今日最新价 - 昨日收盘价) × 今日持仓数量 × 汇率`
    *   `今日总盈亏` = `∑(所有持仓标的今日盈亏)`
*   **快照存储机制**：
    *   每日 16:30 自动抓取行情后，后端必须触发一次快照任务，计算上述「今日总盈亏」，并连同当天的日期存入数据库。

### 3.2 数据源映射 (基于 AKShare)
后端采用 AKShare 免费接口，对应关系如下：
1.  **港股**：`stock_hk_spot_em` (获取最新价/收盘价)
2.  **A股ETF**：`fund_etf_spot_em` 或 `stock_zh_a_spot_em` (获取最新价/收盘价)
3.  **场外基金**：`fund_open_fund_info_em` (获取最新单位净值)
4.  **汇率**：`fx_spot_quote` (获取新浪外汇 HKDCNY 实时汇率)

### 3.3 异常处理与降级策略
1.  **抓取失败降级**：若 AKShare 接口超时或报错，系统自动沿用数据库中记录的**上一个交易日**的价格和汇率，并在前端列表给出「数据陈旧」的UI提示。
2.  **QDII基金延迟**：场外 QDII 基金净值通常有 T-1 或 T-2 延迟，系统按最新能获取到的净值计算，不做强校验。
3.  **分红除权处理**：系统不自动处理分红事件。发生分红时，需用户手动修改持仓成本价，并将分红现金手动加至“现金账户”。

---

## 4. 技术栈与系统架构

### 4.1 技术选型
*   **前端**：Vue3 (Composition API) + Vant 4 UI + Axios
*   **后端**：Python FastAPI (异步轻量) + APScheduler (定时任务) + AKShare
*   **数据库**：SQLite3 (单文件，零配置)
*   **部署环境**：用户自有 Linux 服务器 (Ubuntu/Debian)

### 4.2 部署架构
1.  **反向代理**：Nginx 处理 80/443 端口，配置 SSL 证书。
2.  **后端服务**：Uvicorn 运行 FastAPI，通过 Systemd 托管为后台常驻守护进程。
3.  **前端静态文件**：打包后由 Nginx 直接代理。

---

## 5. UI/UX 与非功能需求

### 5.1 移动端极致适配 (Mobile First)
*   **PWA 支持**：配置 `manifest.json`，支持将网页以 APP 形式添加到手机桌面，隐藏浏览器地址栏，实现沉浸式体验。
